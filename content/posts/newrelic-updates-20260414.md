## はじめに

2026年4月14日のNew Relicアップデートでは、Infrastructure Agent 1.73.1のリリースが行われました。今回のアップデートは、前バージョン1.74.0で発見された不具合への迅速な対応として提供されたロールバックリリースという点が特徴的です。Dockerイメージのビルドプロセス改善、Windowsサポートの更新、および周辺コンポーネントの軽微な更新が含まれており、SREにとって重要な安定性向上が図られています。

## 注目アップデート深掘り

### Infrastructure Agent 1.73.1: 迅速な問題解決とプラットフォーム安定性の向上

今回のInfrastructure Agent 1.73.1リリースは、前バージョン1.74.0で発生した不具合への迅速な対応として実施されたロールバックリリースです。このような迅速な問題解決は、本番環境での監視継続性を重視するSREにとって非常に重要な意味を持ちます。

**SREの日常業務への具体的な影響**

監視設定の観点では、本バージョンによりWindowsプラットフォームでのエージェント安定性が向上し、Windows Serverを運用している環境でのメトリクス収集がより安定します。特に、IISやSQL Serverなどのマイクロソフト製品を監視している場合、エージェントの予期しない停止リスクが軽減されます。

また、Dockerイメージのビルドプロセス改善により、コンテナ化されたエージェントのデプロイメントがより信頼性の高いものになります。ECSタスクやEKSポッドでエージェントを運用している場合、イメージプルエラーや起動失敗のリスクが低減されるでしょう。

**アップグレード手順とベストプラクティス**

EC2インスタンスでの標準的なアップグレード手順：

```bash
# パッケージリストの更新
$ sudo apt update

# Infrastructure Agentのアップグレード
$ sudo apt install newrelic-infra=1.73.1

# サービスの再起動とステータス確認
$ sudo systemctl restart newrelic-infra
$ sudo systemctl status newrelic-infra
```

ECSタスク定義での更新例：

```json
{
  "name": "newrelic-infra",
  "image": "newrelic/infrastructure:1.73.1",
  "essential": false,
  "environment": [
    {
      "name": "NRIA_LICENSE_KEY",
      "value": "your-license-key"
    }
  ]
}
```

> **Note:** Infrastructure Agentは、New RelicのサーバーやクラウドインスタンスからOSレベルのメトリクスやプロセス情報を収集するエージェントソフトウェアです。

nri-dockerとnri-flexコンポーネントの更新により、コンテナメトリクスの収集とカスタムメトリクスの取得がより安定化されているため、これらの機能を活用している環境では特に恩恵を受けられるでしょう。

## SRE視点での活用ポイント

今回のアップデートは、監視基盤の安定性向上に直結する改善が含まれています。特に、前バージョンの不具合への迅速な対応は、New Relicのリリース管理プロセスの信頼性を示しており、SREとして安心してアップデートを適用できる環境が整っています。

**AWS環境での運用における具体的な影響**

EC2インスタンスでの運用では、Windowsベースのワークロードを監視している場合に安定性向上の恩恵を受けられます。ECS環境では、改善されたDockerイメージにより、タスクの起動時間短縮とデプロイメント成功率の向上が期待できます。Lambda環境での影響は限定的ですが、カスタムメトリクスをflex integrationで収集している場合は、より安定した動作を期待できるでしょう。

**すぐに試せるTips**

アップグレード前に、現在のエージェントバージョンとメトリクス収集状況を確認することをお勧めします。`newrelic-infra -version`でバージョンを、New RelicのInfrastructure UIでメトリクスの連続性を確認してからアップグレードを実行しましょう。

**注意点とリスク**

今回はロールバックリリースのため、機能追加よりも安定性に重点が置かれていますが、環境での動作検証は必須です。特に、カスタムconfigurationやflex integrationを多用している環境では、段階的なロールアウトを推奨します。

> **Note:** flex integrationは、New Relicが提供するカスタムメトリクス収集フレームワークで、標準のintegrationでカバーされていないアプリケーションやサービスからメトリクスを収集できます。
