## はじめに

2026年4月13日のNew Relicアップデート情報では、合計6件のアップデートが発表されました。Infrastructure Agentの2つのバージョン更新、Go言語エージェントのセットアップツール、MFA機能の追加、そして実用的なガイド記事2本が公開されています。特に注目すべきは、Goエージェントのセットアップ簡略化とMFA標準機能の追加で、これらは開発・運用効率の向上とセキュリティ強化に大きく貢献します。

## 注目アップデート深掘り

### New Relic Go エージェントの簡単セットアップツール

Goアプリケーションの監視導入において、従来は複雑な設定手順が大きな障壁となっていました。このセットアップツールの登場により、SREの日常業務が大幅に効率化されます。

従来のGoエージェント導入では、依存関係の管理、設定ファイルの作成、環境変数の設定など多くの手順が必要でした。新しいセットアップツールを使用することで、以下のような簡単なコマンドでエージェントを導入できるようになります：

```bash
$ go get github.com/newrelic/go-agent/v3/newrelic
$ newrelic-go-setup --app-name="my-service" --license-key="YOUR_KEY"
```

このツールにより、設定ファイルの自動生成、必要な依存関係のインストール、基本的な計装コードのテンプレート生成が一括で実行されます。SREにとっては、新しいマイクロサービスの監視を数分で開始できるため、障害対応時の緊急監視設定や、開発環境での迅速な検証作業が格段に効率化されます。

### New Relic標準MFA機能の追加

セキュリティ強化の観点から、標準MFA機能の追加は極めて重要なアップデートです。従来、多要素認証を実現するためには外部のSSO サービスや追加のセキュリティ製品導入が必要でした。

新機能では、メールリンク認証方式を採用し、以下の手順でMFAが有効化されます：

1. New Relicコンソールでのアカウント設定画面からMFA有効化
2. 登録メールアドレスに送信される認証リンクのクリック
3. ログイン時の自動的なメールリンク認証の実行

> **Note:** MFA（Multi-Factor Authentication）は、パスワードに加えて第二の認証要素を要求することで、アカウントセキュリティを強化する仕組みです。

SREの運用において、監視ツールへの不正アクセスは重大なセキュリティリスクとなります。インフラストラクチャの監視データや設定情報への不正アクセスを防ぐため、追加コストなしでセキュリティレベルを向上できるこの機能は、特にコンプライアンス要件の厳しい環境で重要な価値を提供します。

## SRE視点での活用ポイント

今回のアップデートは、特にAWS環境でのNew Relic運用において以下の改善をもたらします。

**監視・アラート・ダッシュボードの改善**
Goエージェントのセットアップツールにより、ECSやEKS上のマイクロサービス監視の導入が簡略化され、コンテナ環境での迅速な監視開始が可能になります。Lambda関数のGoランタイム監視についても、同様に導入工数が削減されます。

**セキュリティ運用の強化**
MFA機能により、EC2インスタンスやRDSなどの重要なインフラリソースの監視データへのアクセス制御が強化されます。特に、運用チーム内での権限管理やアクセス監査の要件を満たしやすくなります。

**すぐに試せるTips**
Infrastructure Agent 1.74.0については「Bad release」の警告があるため、1.73.0での運用継続を推奨します。アップグレード前には必ずステージング環境での動作確認を行い、特にWindows環境でのサービスアカウント設定については慎重な検証が必要です。

AWS Systems ManagerやAnsibleを使用したエージェント管理においても、バージョン指定を明示的に行い、意図しないアップグレードを防ぐ設定を推奨します。
