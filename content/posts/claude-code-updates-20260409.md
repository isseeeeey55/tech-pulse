## はじめに

Claude Code の最新リリース v2.1.97 および v2.1.96 がリリースされました。今回のアップデートでは、開発体験の向上とインフラストラクチャ運用の安定化に焦点が当てられています。

v2.1.97 では、フォーカス表示の切り替え、ステータスライン設定の拡張、Git worktree サポート、エージェント稼働インジケーターの追加など、開発効率を大幅に向上させる機能が導入されました。また、パーミッション管理の厳格化、接続の安定性向上、レート制限処理の改善といった、本番運用において重要な堅牢性の向上も図られています。

v2.1.96 では、AWS Bedrock の認証に関する 403 エラーが修正され、クラウドサービス連携の信頼性が大幅に改善されました。これにより、AWS 環境での自動化ワークフローがより安定して動作するようになります。

## 注目アップデート深掘り

### Git Worktree サポートの導入

今回のアップデートで最も注目すべき機能の一つが、Git worktree のサポートです。この機能は、複雑なリポジトリ構成や並行開発において、従来の単一ワーキングディレクトリの制約を解消する画期的な改善です。

Git worktree は、一つのリポジトリから複数のワーキングディレクトリを作成し、異なるブランチを同時に操作できる Git の機能です。従来、機能開発とホットフィックス対応を並行して行う場合、ブランチの切り替えやスタッシュ操作が頻繁に必要でした。Claude Code の worktree サポートにより、これらの煩雑な作業が大幅に軽減されます。

実際の活用例を見てみましょう：

```bash
# メインリポジトリから機能開発用の worktree を作成
$ git worktree add ../feature-auth feature/oauth-integration

# ホットフィックス用の worktree を作成
$ git worktree add ../hotfix hotfix/security-patch

# Claude Code で各 worktree を個別に操作可能
$ cd ../feature-auth
$ claude-code --workspace ./
```

この機能により、SRE やインフラエンジニアは、本番環境への緊急対応と新機能のインフラ構築を同時進行できるようになります。特に、Terraform や Kubernetes マニフェストの管理において、環境ごとの設定変更を並行して進められる利点は計り知れません。

### AWS Bedrock 認証の安定化

v2.1.96 で修正された AWS Bedrock の 403 エラーは、クラウドインフラストラクチャの運用において深刻な問題でした。この修正により、AI 駆動のインフラ自動化やログ解析において、認証エラーによる処理中断が解消されます。

従来、AWS Bedrock との連携では、認証トークンの有効期限やロール切り替えのタイミングで予期しないエラーが発生していました。これにより、夜間のバッチ処理や監視システムからの自動応答が失敗するケースが報告されていました。

修正後の動作例：

```bash
# AWS 認証情報を設定
$ aws configure set region us-east-1
$ aws configure set aws_access_key_id YOUR_ACCESS_KEY
$ aws configure set aws_secret_access_key YOUR_SECRET_KEY

# Claude Code から Bedrock を安定して利用可能
$ claude-code --bedrock-model anthropic.claude-3-sonnet
```

この改善により、インシデント対応時の AI アシスタント活用や、ログ解析の自動化において、信頼性の高い運用が実現されます。特に、24時間365日の監視体制において、認証エラーによるサービス中断リスクが大幅に軽減されました。

## 実用的な活用ポイント

今回のアップデートは、日常的な開発ワークフローに多くの改善をもたらします。特に注目すべきは、ステータスライン設定の拡張です。この機能により、開発環境の状態を常に把握できるようになり、問題の早期発見と迅速な対応が可能になります。

エージェント稼働インジケーターの追加も実用的な改善です。バックグラウンドで動作するプロセスの状態がリアルタイムで確認できるため、処理の完了を待つ時間や、フリーズしたプロセスの特定が容易になります。これは特に、大規模なコードベースでの作業や、時間のかかる解析処理において威力を発揮します。

SRE の観点では、Cedar policy ファイルのシンタックスハイライト対応が重要です。AWS の IAM ポリシーや、アクセス制御ルールの可読性が向上し、セキュリティポリシーのレビューや監査作業の効率が大幅に改善されます。また、パーミッション管理の厳格化により、本番環境での誤操作リスクが軽減され、より安全な運用が実現されます。

すぐに試せる Tips として、新しいフォーカス表示機能を活用してください。複数のファイルやタスクを同時に扱う際に、重要な作業に集中できる環境を素早く構築できます。これにより、障害対応時の集中力維持や、コードレビューの精度向上が期待できます。
