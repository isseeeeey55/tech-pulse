## はじめに

Claude Code v2.1.90 がリリースされました。今回のアップデートでは、インタラクティブレッスン機能の追加、環境変数の拡張対応、そして多数のバグ修正と性能改善が実施されています。特にオフライン環境での安定性向上とエッジケースでの使用性改善に重点が置かれており、企業環境での実用性が大幅に向上しています。

主な変更点として、マーケットプレイス設定の柔軟な管理機能、長大な会話セッションでの応答性改善、PowerShellツールのセキュリティ強化などが挙げられます。これらの改善により、SREやインフラエンジニアの日常業務において、より安全で効率的なワークフローの構築が可能になります。

## 注目アップデート深掘り

### インタラクティブレッスン機能の追加

今回のリリースで最も注目すべき機能の一つが、インタラクティブレッスン機能の導入です。この機能は、チーム内でのノウハウ共有や新人エンジニアのオンボーディングを大幅に効率化することを目的としています。

従来、複雑な運用手順やトラブルシューティング手順の共有は、ドキュメントやWikiに依存していましたが、実際の操作環境と乖離することが多く、学習効果に限界がありました。インタラクティブレッスン機能では、実際のコマンド実行や設定変更を含む対話的な学習体験を提供できます。

```bash
# レッスンの作成例
$ claude-code lesson create --name "kubernetes-troubleshooting" \
  --description "本番環境でのPod障害対応手順" \
  --interactive

# レッスン内でのステップ定義
$ claude-code lesson add-step \
  --lesson kubernetes-troubleshooting \
  --title "Pod状態の確認" \
  --command "kubectl get pods -n production" \
  --expected-output "Running状態以外のPodを特定"
```

この機能により、SREチームでは実践的な運用手順書を作成し、チームメンバーが実際の環境で安全に学習できる環境を構築できます。特に、障害対応手順のような高度な技術知識が必要な領域では、座学では得られない実践的なスキルの習得が可能になります。

### 環境変数の拡張とオフライン環境対応

もう一つの重要な改善点は、環境変数による詳細な制御機能の拡張と、オフライン環境でのマーケットプレイス設定の柔軟な管理機能です。企業環境では、セキュリティポリシーにより外部ネットワークへのアクセスが制限されることが多く、これまでのバージョンでは設定の自由度に制約がありました。

新しいバージョンでは、以下のような環境変数による細かな制御が可能になりました：

```bash
# オフライン環境でのマーケットプレイス設定
export CLAUDE_CODE_MARKETPLACE_MODE=offline
export CLAUDE_CODE_LOCAL_REGISTRY=/opt/claude-code/local-registry
export CLAUDE_CODE_TOOL_CACHE_TTL=86400

# セキュリティレベルの調整
export CLAUDE_CODE_SECURITY_LEVEL=strict
export CLAUDE_CODE_POWERSHELL_EXECUTION_POLICY=RemoteSigned
export CLAUDE_CODE_ALLOWED_DOMAINS="*.internal.company.com,localhost"
```

Before では、オフライン環境でツールの追加や更新を行う際に複雑な回避策が必要でしたが、After では環境変数による直接的な制御により、企業のセキュリティポリシーに準拠しながらも柔軟な運用が実現できます。これにより、閉域網環境での開発やセキュアな本番環境でのメンテナンス作業が大幅に効率化されます。

## 実用的な活用ポイント

今回のアップデートは、日常の開発ワークフローに以下のような具体的な改善をもたらします。

**CI/CDパイプラインでの活用**では、環境変数の拡張により、異なる環境（開発、ステージング、本番）での Claude Code の動作を統一的に制御できるようになりました。特に、Docker コンテナ内での実行時にキャッシュ設定やタイムアウト値を環境に応じて調整することで、パフォーマンスとリソース使用量の最適化が図れます。

**チーム運用の標準化**においては、インタラクティブレッスン機能を活用することで、属人的になりがちな運用ノウハウを体系的に共有できます。例えば、データベースのパフォーマンスチューニング手順や、Kubernetes クラスターの health check 手順などを、実際のコマンド実行を含む形で標準化し、チーム全体のスキルレベル向上に寄与します。

**セキュリティ要件への対応**では、PowerShell ツールの強化されたセキュリティチェックにより、Windows 環境でのインフラ自動化がより安全に実行できるようになりました。これは特に、Active Directory の管理や Windows Server の設定変更など、権限昇格を伴う作業において重要な改善です。

すぐに試せる Tips として、まず `CLAUDE_CODE_DEBUG=verbose` 環境変数を設定してログの詳細度を上げ、既存のワークフローでの動作状況を確認することをお勧めします。これにより、パフォーマンスボトルネックの特定や、新機能の効果測定が容易になります。

## まとめ

Claude Code v2.1.90 は、企業環境での実用性向上に重点を置いた堅実なアップデートとなっています。特に、オフライン環境対応とセキュリティ強化により、これまで導入が困難だった厳格なセキュリティポリシーを持つ組織でも活用しやすくなりました。

インタラクティブレッスン機能の追加は、技術的なノウハウの属人化という長年の課題に対する有効なソリューションを提供しており、チーム全体の技術レベル向上とナレッジマネジメントの効率化に大きく寄与すると期待されます。

パフォーマンス面での改善も地味ながら重要で、大規模な運用環境での長時間セッションにおいても安定した動作が期待できるようになりました。これらの改善により、Claude Code は開発ツールとしてだけでなく、本格的な運用ツールとしての地位を確立しつつあると言えるでしょう。
