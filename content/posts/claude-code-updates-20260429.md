## はじめに

2026年4月29日、Claude Code の v2.1.122 および v2.1.121 が相次いでリリースされました。今回のアップデートでは、**MCP サーバーの安定性向上**と**クラウドプロバイダー連携の強化**が大きな柱となっています。

主な変更点は以下の通りです：

- **MCP サーバーの自動リトライ機能**の実装により、接続の安定性が向上
- **Bedrock サービスティア選択機能**の追加により、AWS 環境での柔軟な運用が可能に
- **PR URL によるセッション検索対応**で、コードレビューワークフローが改善
- **Vertex AI の mTLS 認証対応**により、Google Cloud 環境でのセキュリティが強化
- **OpenTelemetry のログ出力**が数値型で正確に記録されるよう修正
- **ツール検索の最適化**と**権限設定の簡素化**により、開発体験が向上

これらのアップデートは、エンタープライズ環境での運用安定性とセキュリティを重視した内容となっており、特に複数のクラウドプロバイダーを活用する組織にとって有益な機能強化が含まれています。

## 注目アップデート深掘り

### MCP サーバーの自動リトライ機能と重複検出の改善

今回のリリースで最も注目すべきは、**MCP サーバーの安定性向上**に関する一連の改善です。v2.1.121 では自動リトライ機能が実装され、v2.1.122 では重複検出機能が強化されました。

MCP（Model Context Protocol）サーバーは、Claude Code が外部ツールやデータソースと連携する際の中核的なコンポーネントです。しかし、ネットワークの一時的な不安定性やサーバー側の遅延により、接続が失敗するケースが実運用では避けられません。従来は接続失敗時に手動での再試行が必要でしたが、今回の自動リトライ機能により、一時的な障害に対して自動的に復旧を試みるようになりました。

この変更がもたらす実質的なメリットは、**長時間稼働するセッションでの信頼性向上**です。特に CI/CD パイプラインに組み込んだ自動化タスクや、夜間バッチ処理など、人間の介入が難しいシナリオでの安定性が大幅に改善されます。

さらに v2.1.122 では、MCP サーバーの重複検出機能が改善されました。これにより、同一のサーバーが複数回登録されることを防ぎ、設定ミスによるリソースの無駄遣いや予期しない動作を回避できます。複数の設定ファイルや環境変数から MCP サーバーを読み込む場合に、特に有効な改善と言えるでしょう。

### Bedrock サービスティア選択機能の追加

AWS Bedrock を利用しているユーザーにとって重要なアップデートが、**サービスティア選択機能**の追加です。この機能により、Bedrock の異なるサービスティアを明示的に選択できるようになりました。

AWS Bedrock は、用途に応じて複数のサービスティアを提供しており、それぞれコスト・パフォーマンス・利用制限が異なります。従来は Claude Code 側でティアを指定する方法が限定的でしたが、今回の機能追加により、ワークロードの特性に応じた最適なティアを選択できるようになりました。

この変更が重要な理由は、**コスト最適化とパフォーマンスのバランス調整**が可能になる点です。例えば、開発環境やテスト用途では低コストなティアを使用し、本番環境や高スループットが必要なタスクでは高性能なティアを選択する、といった運用が実現できます。特に SRE やインフラエンジニアにとって、環境ごとの予算管理やリソース配分の柔軟性が向上する意義は大きいでしょう。

また、組織のセキュリティポリシーやコンプライアンス要件によっては、特定のティアの使用が義務付けられている場合もあります。明示的な選択肢が提供されることで、そうした要件への対応も容易になります。

## 実用的な活用ポイント

今回のアップデートを日常の開発ワークフローに活かすためのポイントをいくつか紹介します。

**PR URL でのセッション検索**は、コードレビュープロセスの効率化に直結します。プルリクエストに関連する過去のセッションを素早く検索できるため、レビュー時の文脈把握やトラブルシューティングが迅速化します。特に大規模なチームで複数の PR を同時進行している場合、この機能により関連情報へのアクセスが容易になるでしょう。

**Vertex AI の mTLS 認証対応**は、Google Cloud 環境で Claude Code を運用している組織にとって重要なセキュリティ強化です。相互 TLS 認証により、クライアントとサーバー双方の認証が強化され、よりセキュアな通信が実現します。金融やヘルスケアなど、厳格なセキュリティ要件がある業界では特に有益でしょう。

**OpenTelemetry のログ出力改善**により、監視とトラブルシューティングの精度が向上しました。数値型データが正確に記録されることで、メトリクスの集計や異常検知の精度が高まります。Grafana や Datadog などの監視ツールと連携している場合、より正確なダッシュボードやアラート設定が可能になります。

**ツール検索の最適化**と**権限設定の簡素化**は、日々の開発体験を地味ながら確実に向上させます。必要なツールへの迅速なアクセスと、シンプルな権限管理により、開発者はより本質的なタスクに集中できるようになります。

## まとめ

v2.1.122 および v2.1.121 のリリースは、**安定性・セキュリティ・運用性**の三軸での着実な改善が特徴です。派手な新機能の追加よりも、日々の運用で直面する課題への丁寧な対応が目立ちます。

特に MCP サーバー周りの改善は、Claude Code をプロダクション環境で安心して運用するための基盤強化と言えます。自動リトライと重複検出により、長時間稼働や複雑な設定環境でのトラブルが減少するでしょう。

また、AWS Bedrock と Google Cloud Vertex AI という二大クラウドプロバイダーへの対応強化は、マルチクラウド戦略を採用する組織にとって朗報です。各クラウドの特性を活かした最適な運用が可能になります。

細かな UI 改善やバグ修正も多数含まれており、総合的な開発体験の向上が期待できるリリースと言えるでしょう。既存ユーザーは早めのアップデートを検討する価値があります。
